# Contributing

Thank you for your interest in contributing to Paperless.

## Getting started

1. Fork the repository.
2. Create a feature branch from `main`.
3. Install the dependencies for the client and server.
4. Make your changes and test them locally.
5. Open a pull request with a clear summary of your work.

## Guidelines

- Keep changes focused and well documented.
- Follow the existing code style and architecture.
- Include tests or validation where appropriate.
- Be respectful and constructive in discussions.

## Reporting issues

Please open an issue with as much detail as possible, including steps to reproduce and expected behavior.
